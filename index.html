IDAM — Identity & Access Management Tickets

Track provisioning, deprovisioning, role changes, and access requests. Follows NIST SP 1800-2 IdAM workflow stages.

IDAM Ticket Lifecycle Reference ▼
1
Identity Request
User/system submits access request. HR/manager approval required. Identity proofing documentation gathered.
2
Provisioning
Digital identity created in Active Directory / IdP. Credentials issued (badge, certs, logical access). Least-privilege applied.
3
Access Assignment
Role-based access controls configured. SSO/MFA enrolled. Resources mapped per job function.
4
Work Performed
Technical implementation documented. Evidence collected (screenshots, AD exports, logs).
5
Verification & Closure
Ticket reviewed, work performed document attached, closed in system. Audit trail complete.
⚡ Common Issues
Orphaned accounts not deprovisioned Missing MFA enrollment Excessive privilege granted AD sync latency No work performed doc attached Role not matching job function Missing manager approval

ServiceNow — Tickets & Active Directory

Incident management, AD on-prem issues, change requests, and service catalog items.

ServiceNow Ticket Lifecycle & AD On-Prem Reference ▼
1
Ticket Logged
Incident/Request raised in ServiceNow. Categorized (Incident, Change, Service Request, Problem). SNOW ticket ID assigned.
2
Triage & Assignment
Priority assessed (P1-P4). Assigned to appropriate team. SLA clock starts. AD issues routed to identity team.
3
Investigation
Root cause analysis. AD logs reviewed, replication status checked, GPO conflicts investigated. Work notes documented.
4
Resolution
Fix applied. AD object corrected/synced. Changes tracked in SNOW change record. Evidence screenshots captured.
5
Close & Post-Incident
Ticket closed. Post-incident review for P1/P2. Knowledge article created if new issue. SLA performance recorded.
⚡ Common AD On-Prem Issues
AD replication failurePassword sync loopLocked accountGPO not applyingKerberos pre-auth failureDNS resolution failureSPN conflictDomain trust brokenSYSVOL not replicating

UAR — User Access Review

Full lifecycle tracking: Scoping → Inventory → Certification → Remediation → Documentation. Per NIST AC-2 & AC-6 requirements.

UAR 6-Phase Process Reference ▼
1
Scoping & Prioritization
Define which systems are in scope. Prioritize: privileged accounts (quarterly), standard users (annual).
2
Access Inventory
Generate current access list: usernames, group memberships, last login, roles/job functions.
3
Certification
Route to managers/system owners for approve/revoke decisions. Document rationale for each.
4
Risk Assessment
Flag excessive privileges, orphaned accounts, former employees. Assess privilege creep.
5
Remediation
Revoke/modify access per decisions. Create SNOW tickets for each remediation. Verify changes.
6
Documentation
Generate audit-ready report: scope, methodology, findings, remediation actions, approver signatures.
⚡ Common UAR Issues
Reviewer non-response / escalation neededFormer employee access not revokedPrivilege creep detectedOrphaned accountsMissing audit trail documentationAccess data stale at time of reviewNo remediation tickets created

FedRAMP — Authorization & Compliance

Track authorization journey: Preparation → 3PAO Assessment → ATO → Continuous Monitoring (ConMon). NIST 800-53 controls.

FedRAMP Authorization Phases & ConMon Reference ▼
1
Preparation
Define authorization boundary & FIPS 199 categorization. Develop SSP, POA&M, architecture diagrams.
2
Readiness Assessment
3PAO conducts RAR. Validates technical capabilities, security requirements maturity, ConMon readiness.
3
Authorization Assessment
3PAO develops SAP. Tests controls against SSP claims. Produces SAR. POA&M updated with findings.
4
ATO Issuance
Agency AO reviews full package (SSP, SAP, SAR, POA&M). Issues Authority to Operate (ATO).
5
ConMon (Ongoing)
Monthly: vulnerability scans, POA&M updates. Annual: 3PAO independent assessment, penetration testing.
⚡ Common FedRAMP Issues
SSP documentation inconsistenciesAuthorization boundary undefined for 3rd-party toolsMissed monthly ConMon deliverablePOA&M finding past SLA deadlineControl not implemented per SSP claimVulnerability scan not covering full inventoryIncident not reported within 1-hour windowAnnual 3PAO assessment overdue
